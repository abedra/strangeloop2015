Setup

Build and install hiredis

            
$ cd repsheet/hiredis
$ make
$ sudo PREFIX=/usr make install
$ sudo ldconfig # if you are on linux

Build and install librepsheet

            
$ cd repsheet/librepsheet
$ ./autogen.sh
$ ./configure --prefix=/usr
$ make
$ make check # all tests should pass
$ sudo make install
$ sudo ldconfig # if you are on linux

Build and install NGINX with the repsheet module

            
$ cd repsheet/repsheet-nginx
$ bundle # if you want to run the tests
$ script/bootstrap
$ rake # all tests should pass

Repsheet

User behavior

Our web systems are not fortresses

And our responses shouldn't be static

Example

	    
location / {
  # block one workstation
  deny    192.168.1.1;
  # allow anyone in 192.168.1.0/24
  allow   192.168.1.0/24;
  # drop rest of the world
  deny    all;
}

This doesn't scale

And it makes operations much harder

It also doesn't allow you to change your response according to the actor's behavior

Repsheet gives you more fluid options for dealing with bad actors

Let's start with the configuration

Options

	    
repsheet on;                          # main or location conf
repsheet_user_lookup on;              # main conf
repsheet_ip_lookup on;                # main conf

repsheet_redis_host localhost;        # main conf
repsheet_redis_port 6379; # main conf
repsheet_redis_connection_timeout 5;  # main conf
repsheet_redis_read_timeout 10;       # main conf

repsheet_proxy_headers on;            # main conf
repsheet_user_cookie "user";          # main conf

A full configuration is provided in the repsheet-nginx repository
